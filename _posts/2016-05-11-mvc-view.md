## 概念

你一定看过一个`controller`的`action`方法返回`ActionResult`，或者`ViewResult`方法，
并且会在客户端调用视图，这一章我们将集中弄清这些知识，我们将展示MVC框架的`view engines`
怎么处理`ViewResult`，并且包含怎样创建自定义的`view engines`





### 创建自定义`view engines`

在大多数的项目中，可能不需要创建自定义`view engines`，MVC框架包含两种功能齐全并且很好测试的`view engines`

>* `Razor engine`，它有简单而优雅的语法
>* asp.net `<%...%>标记语法，MVC4之前使用的语法

*`IViewEngine`接口*

```c#
public interface IViewEngine { 
	ViewEngineResult FindPartialView(ControllerContext controllerContext,  
		string partialViewName, bool useCache); 

	ViewEngineResult FindView(ControllerContext controllerContext,              
	string viewName, string masterName, bool useCache); 

	void ReleaseView(ControllerContext controllerContext, IView view); 
} 
```

`view engine`作用是为`view`转换`ViewEngineResult`对象，这里有2个方法`FindView`
和`FindPartialView`，获得`controller`传递过来的参数

MVC框架支持实现了`ControllerActionInvoker`接口的`view engine`，它是一个内置的实现了`IActionInvoker`
的接口

当一个视图被请求，`ViewEngineResult`允许MVC框架的`view engine`响应

```c#
public class ViewEngineResult { 
	public ViewEngineResult(IEnumerable<string> searchedLocations) { 
		if (searchedLocations == null) { 
			throw new ArgumentNullException("searchedLocations"); 
		} 
		SearchedLocations = searchedLocations; 
	} 

	public ViewEngineResult(IView view, IViewEngine viewEngine) {             
		if (view == null) { 
			throw new ArgumentNullException("view");
		} 
		if (viewEngine == null) { 
			throw new ArgumentNullException("viewEngine");
		} 
		View = view; 
		ViewEngine = viewEngine; 
	} 

	public IEnumerable<string> SearchedLocations { get; private set; } 
	public IView View { get; private set; } 
	public IViewEngine ViewEngine { get; private set; } 
} 
```

如果`view engine`能够为请求提供视图，你可以创建一个`ViewEngineResult`
构造函数

```c#
public ViewEngineResult(IView view, IViewEngine viewEngine)
```

这个构造函数的参数是一个的`IView`接口实现和一个`view engine`，如果你的
`view engine`不能够提供视图，你可以使用下面的构造函数

```c#
public ViewEngineResult(IEnumerable<string> searchedLocations)
```

最后一个`view engine`系统的构建块是`IView`接口

```c#
public interface IView { 
	void Render(ViewContext viewContext, TextWriter writer); 
} 
```

我们经过一个`IView`实现`ViewEngineResult`对象的构造函数，MVC框架调用
`Render`方法，`ViewContext`参数传递从客户端获得的参数信息和从`action`方法
输出的信息

### 创建自定义的`IView`

```js
public class DebugDataView:IView
{
	public void Render(ViewContext viewContext, TextWriter writer)
	{
		Write(writer, "---Routing Data---");
		foreach (string key in viewContext.RouteData.Values.Keys)
		{
			Write(writer, "Key: {0}, Value: {1}", key, viewContext.RouteData.Values[key]);
		}

		Write(writer, "---View Data---");
		foreach (string key in viewContext.ViewData.Keys)
		{
			Write(writer, "Key: {0}, Value: {1}", key, viewContext.ViewData[key]);
		} 
	}

	private void Write(TextWriter writer, string template, params object[] values)
	{
		writer.Write(string.Format(template, values) + "<p/>");
	} 
}
```

### 实现`IViewEngine`接口

记住,视图引擎的目的是产生一个`ViewEngineResult`对象,它包含一个`IView`或者一个寻找一个合适的视图
的列表

*自定义的`IViewEngine`实现*

```c#
public class DebugDataViewEngine : IViewEngine 
{
	public ViewEngineResult FindPartialView(ControllerContext controllerContext, 
		string partialViewName, bool useCache)
	{
		return new ViewEngineResult(new string[] { "No view (Debug Data View Engine)" });
	}

	public ViewEngineResult FindView(ControllerContext controllerContext, 
		string viewName, string masterName, bool useCache)
	{
		if (viewName == "DebugData")
		{
			return new ViewEngineResult(new DebugDataView(), this);
		}
		else
		{
			return new ViewEngineResult(new string[] { "No view (Debug Data View Engine)" });
		} 
	}

	public void ReleaseView(ControllerContext controllerContext, IView view)
	{
		//do nothing
	}
}
```

当请求这个视图，我们将返回自定义的`IView`实现类实例

```c#
return new ViewEngineResult(new DebugDataView(), this);
```

如果我们实现一系列的`view engine`，可以像下面这样创建

```c#
return new ViewEngineResult(new string[] { 
	"No view (Debug Data View Engine)" 
});
```

### 注册自定义的`View Engine`

我们把 `view engines`注册到 `Global.asax`的`Application_Start`

```c#
public class MvcApplication : System.Web.HttpApplication
{
	protected void Application_Start()
	{
		AreaRegistration.RegisterAllAreas();
		ViewEngines.Engines.Add(new DebugDataViewEngine());
	}
}
```

`ASPX`等，如果你只想使用自定义的`view engine`，可以清除其他`view engine`

```c#
 public class MvcApplication : System.Web.HttpApplication
{
	protected void Application_Start()
	{
		ViewEngines.Engines.Clear();  
		ViewEngines.Engines.Add(new DebugDataViewEngine());
	}
} 
```

#### 明白`Razor View`

`Razor view`编译应用程序的视图来提高性能，视图转换成`c#`的类，然后编译，
这就是为什么你能在视图里面方便的写`c#`代码，视图只有在程序启动的时候才编译，

视图文件生成的`c#`类代码文件被写入磁盘,然后编译。视图生成的类是这样的

```c#
public class _Page_Views_Home_Index_cshtml : System.Web.Mvc.WebViewPage<string[]> { 
        public _Page_Views_Home_Index_cshtml() { 
        } 
 
        public override void Execute() { 
            ViewBag.Title = "Index"; 
            WriteLiteral("\r\n\r\nThis is a list of fruit names:\r\n\r\n"); 
 
            foreach (string name in Model) {                 
				WriteLiteral("    <span><b>"); 
                Write(name);                 
                WriteLiteral("</b></span>\r\n"); 
            } 
        } 
    } 
}
```

首先，这个类派生于`WebViewPage<T>`，这就是为什么`views`支持强类型
，在执行方法里面，`@`语法的内容都被编译成`c#`结构，`HTML`元素被`WriteLiteral`方法处理，

### 配置`View`搜索地址

`Razor View Engine` 如果查找`Home`控制器的`Index action`的视图，跟随下面的规则

>* ~/Views/Home/Index.cshtml
>* ~/Views/Home/Index.vbhtml
>* ~/Views/Shared/Index.cshtml
>* ~/Views/Shared/Index.vbhtml

`Razor`不是真的在硬盘上查找文件，因为它们已经编译成`c# classes`了，
`Razor`去查找那些代表`views`的编译的`class`,

你可以改变`Razor`的查找方式，创建一个`RazorViewEngine`的子类，
这个类将实现`IViewEngine`

*`Razor`的搜索属性*

属性					|描述				|默认值
ViewLocationFormats<br />MasterLocationFormats<br />PartialViewLocationFormats<br />  		|`views`,`partial views`以及`layouts`的路径		|~/Views/{1}/{0}.cshtml, <br />~/Views/{1}/{0}.vbhtml,<br /> ~/Views/Shared/{0}.cshtml,<br /> ~/Views/Shared/{0}.vbhtml 
AreaViewLocationFormats<br />AreaMasterLocationFormats<br />AreaPartialViewLocationFormats<br /> | `views`,`partial views`以及`layouts`的路径	|~/Areas/{2}/Views/{1}/{0}.cshtml, <br />~/Areas/{2}/Views/{1}/{0}.vbhtml,<br /> ~/Areas/{2}/Views/Shared/{0}.cshtml, <br />~/Areas/{2}/Views/Shared/{0}.vbhtml 

在上面的表格中，`{0}`,`{1}`,`{2}`占位符的意思

>* `{0}`表示`view`的名字
>* `{1}`表示`control`的名字
>* `{2}`表示`area`的名字

为了改变默认的搜索地址，我们创建一个新的类，派生于`RazorViewEngine`

我们在`ViewLocationFormats`里面设置了新值，我们已经改变了搜索的位置,默认从`View/Common`
里面查找，而不是`View/Shared`，我们把这个功能注册到`Application_Start`

```c#
public class MvcApplication : System.Web.HttpApplication
{
	protected void Application_Start()
	{
		ViewEngines.Engines.Clear();
		ViewEngines.Engines.Add(new CustomLocationViewEngine()); 
	}
}
```

我们`Views`文件夹下面创建`Common`文件夹，创建`List.cshtml`视图，然后创建`List action`方法来
测试下，是否能找到`~/Views/Common/List.cshtml`视图，结果是可以找到的

### 给`Razor View`添加动态内容

视图的主要功能是允许你显示领域模型的用户界面，你需要有一个能添加动态内容的视图，在运行时生成动态内容，你可以使用下面的方法添加动态内容

*给`view`添加动态内容*

技术				|什么时候使用
`Inline code`		|使用小的、独立的部分视图逻辑，例如`if`、`foreach`，这是最基本的创建动态视图的方法
`HTML helper`方法	|用来生成`html`元素或小的集合，通常基于视图模型或视图数据，
`Sections`			|在特定位置插入的部分内容的布局
`Partial views`		|共享视图的标记，`Partial view`能包含`inline code`、`Html helper`方法,不能够调用`action`方法，所以不能有业务逻辑
`Child actions`		|用于创建可重用的UI控件或包含业务逻辑的小部件，当你使用`Child actions`,它调用`action`方法，渲染`view`，并且把结果注入到`response stream`

### 使用`Sections`

`Razor engine`支持`sections`概论，允许提供区域的内容布局，
`Razor sections`可以控制哪些`layout`可以插入到`view`
