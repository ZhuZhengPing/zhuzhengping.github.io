## 概念

每个程序都要使用资源，在面向对象的环境中，每个类型都代表可供程序使用的一种资源。要使用这些资源，必须为代表资源的类型分配内存。

>* 调用IL指令 newobj,为代表资源的类型分配内存（一般使用C# new 操作符来完成）。
>* 初始化内存，设置资源的初始化状态并使资源可用。类型的实例构造器负责设置初始状态
>* 访问类型的成员来使用资源(有必要可以重复)。
>* 摧毁资源的状态以进行清理
>* 释放内存。垃圾回收器独自负责这一步。









#### 从托管堆分配资源

对于托管堆，分配对象只需在指针上加上一个值，速度相当快。由于托管堆在内存中连续分配这些对象，所以会因为引用的“局部化”而获得性能上的提升。这意味着
进程的工作集会非常小，应用程序只需使用很少的内存，从而提高了速度。。， 开门， 额·  

但是内存不可能无限，所以CLR通过称为“垃圾回收”（GC）的技术删除堆中不需要的对象。


#### 垃圾回收算法

应用程序调用 new 操作符创建对象时，可能没有足够地址空间来分配该对象。发现空间不够，CLR就执行垃圾回收。

CLR开始GC时，首先暂停进程中的所有线程。然后,CLR进入GC的标记阶段。在这个阶段，CLR遍历堆中的所有对象，将同步块索引字段中的一位设为0。这表明所有对象都应删除。
然后，CLR检查所有活动根，查看它们引用了哪些对象。任何根引用了堆上的对象，CLR都会标记那个对象，也就是将该对象的同步块索引为设置为1.已标记的对象不能被
垃圾回收，CLR对堆中已标记的对象进行“乾坤大挪移”，压缩所有幸存下来的对象，使它们占用连续的内存空间。

#### 垃圾回收和调试

一旦根离开作用域，它引用的对象就会变成可回收，如下面的代码

```c#
public static class Program{
	public static void Main(){
		Timer t = new Timer(TimerCallback,null,0,2000);
		
		Console.ReadLine();
	}
	
	private static void TimerCallback(object o){
		Console.WriteLine("In TimerCallback: "+DateTime.Now);
		
		// 出于演出目的，强制执行一次垃圾回收
		GC.Collect();
	}
}
```

在命令行上，不用任何特殊编译器开关编译代码。运行可执行文件，会发现 TimerCallback 方法只被调用一次！

回收开始时，垃圾回收器检查应用程序的根，发现在初始化后，Main 方法再也没有用过变量t，就被回收了。

可试着像下面这样修改 Main 方法：

```c#
public static void Main(){
	public static void Main(){
		Timer t = new Timer(TimerCallback,null,0,2000);
		
		Console.ReadLine();
		
		// 在 ReadLine 之后引用 t （会被优化掉）
		t=null;
	}
}
```

执行上面的代码，TimerCallback 方法仍然只被调用一次。将局部变量或参数设置为null,等价于不引用该变量。

下面才是 Main 方法的正确修改方式：

```c#
public static void Main(){
	public static void Main(){
		Timer t = new Timer(TimerCallback,null,0,2000);
		
		Console.ReadLine();
		
		t.Dispose();
	}
}
```
