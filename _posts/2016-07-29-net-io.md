## 概念

在.NET Framework中，System.IO  命名空间主要包括基于文件(和基于内存)的输入输出(I/O)服务的相关基础类库。和其他命名空间一样，System.IO 定义了一系列类、接口、枚举和委托。他们大多数包含在 mscorlib.dll中，另外有一部分 System.IO 命名空间的成员则包含在 System.dll 程序集中。






*System.IO*命名空间的主要成员

非抽象 I/O 类类型						|作用
BinaryReader 和 BinaryWriter			|这两个类型能够以二进制值存储和读取基本数据类型(整形、布尔型、字符串型和其他类型)
BufferedStream							|这个类型为字节流提供了临时的存储空间，可以以后提交
Directory 和 DirectoryInfo				|这两个类型用来操作计算机的目录结构。Directory 类型主要的功能通过静态方法实现。DirectoryInfo 类型则通过一个有效的对象引用来实现类似功能
DriveInfo								|提供计算机驱动器的详细信息
File 和 FileInfo						|这两个类型用来操作计算机上的一组文件。File 类型主要的功能通过静态成员实现，FileInfo 类型则通过一个有效的对象引用来实现类似功能
FileStream								|这个类型实现文件随机访问(比如寻址能力)
FileSystemWatcher						|这个类型监控对指定外部文件的更改
MemoryStream							|这个类型实现对内存(而不是物理文件)存储的流数据的随机访问
Path									|这个类型对包含文件或目录路径信息的 System.String 类型执行操作。这些操作是与平台无关的
StreamWriter 和 StreamReader			|这两个类型用来在文件中存储或获取文本信息。不支持随机文件访问
StringWriter 和 StringReader			|和 StreamWriter/StreamReader 类型差不多，这两个类型同样和文本信息打交道，不同的是基层的存储器是字符串缓冲区而不是物理文件

### Directory 和 File 类型

System.IO 提供了4个类型来实现对单个文件和计算机目录结构的操作。前两个类型 Directory 和 File 通过各种静态成员实现建立、删除、复制和移动操作。与之紧密关联的 FileInfo 和 DirectoryInfo 类型则通过实例级方法来实现类似的功能(因此必须用 new 关键字分配它们)。

<img src="http://ww3.sinaimg.cn/small/006dag38jw1f6bvtjujr9j30gv08haao.jpg" />

一般来说，FileInfo 和 DirectoryInfo 是获取文件或目录细节(如创建时间、读写能力等)更好的方式，因为它们的成员往往会返回强类型的对象。相反，Directory 和 File 类往往会返回简单字符串值而不是强类型对象。

### FileSystemInfo 抽象基类

DirectoryInfo 和 FileInfo 类型实现了许多 FileSystemInfo 抽象基类的行为。大部分 FileSystemInfo 类成员的作用是用来获取指定文件或目录的一般特性。

*FileSystemInfo*

属性			|作用
Attributes		|获取或设置与当前文件关联的特性，由 FileAttributes 枚举表示(例如，只读、加密、隐藏或压缩文件或目录)
CreationTime	|获取或设置当前文件或目录的创建时间
Exists			|用来判断指定文件或目录是否存在的值
Extension		|获取文件的扩展名
FullName		|获取目录或文件的完整路径
LastAccessTime	|获取或设置上次访问当前文件或目录的时间
LastWriteTime	|获取或设置上次写入当前文件或目录的时间
Name			|获取当前文件或目录的名称

FileSystemInfo 类型还定义了 Delete()方法，该操作由派生类型从硬盘中删除指定文件或目录来实现。同样，在获取文件特性前使用 Refresh()方法能确保当前文件(或目录)的统计信息是最新的。

### 使用 DirectoryInfo 类型

它包含一组用来创建、移动、删除和枚举所有目录/子目录的成员。

*DirectoryInfo 类型的主要成员*

Create()和CreateSubdirectory()	|按照路径名建立一个目录(或一组子目录)
Delete()						|删除一个目录和它的所有内容
GetDirectories()				|返回一个表示当前目录中所有子目录的 DirectoryInfo 对象数组
GetFiles()						|返回 FileInfo 对象数组，表示指定目录下的一组文件
MoveTo							|将一个目录及其内容移动到一个新的路径
Parent							|获取指定路径的父目录
Root							|获取路径的根部分

我们首先使用 DirectoryInfo 类型指定一个特别的目录路径作为构造函数的参数。如果需要访问当前应用程序的目录的话，可以使用“.”符合。

```c#
// 绑定到当前的应用程序目录
DirectoryInfo dir1 = new DirectoryInfo(".");

// 使用 verbatim字符串绑定到 D:\Test
DirectoryInfo dir2 = new DirectoryInfo(@"D:\Test");
```

在第二个例子中，必须确保传入构造函数的路径(C:\Windows)是在物理计算机上存在的。然而如果试图使用一个不存在的目录，系统就会引发 System.IO.DirectoryNotFoundException 异常。因此，如果指定了一个尚未创建的目录的话，在对目录进行操作前首先需要调用 Create()方法。

```c#
// 绑定到一个不存在的目录，然后创建它
DirectoryInfo dir3 = new DirectoryInfo(@"D:\MyTest");
dir3.Create();
```

创建了 DirectoryInfo 对象后，就能使用任何一个派生自 FileSystemInfo 的属性来获取底层目录的内容。为了演示，创建一个控制台应用程序

```c#
static void Main(string[] args)
{

	ShowWindowsDirectoryInfo();

	Console.ReadLine();
}

static void ShowWindowsDirectoryInfo()
{
	// 转储目录信息
	DirectoryInfo dir = new DirectoryInfo(@"c:\Windows");
	Console.WriteLine("FullName: {0}",dir.FullName);
	Console.WriteLine("Name: {0}",dir.Name);
	Console.WriteLine("Parent: {0}",dir.Parent);
	Console.WriteLine("Creation: {0}",dir.CreationTime);
	Console.WriteLine("Attributes: {0}",dir.Attributes);
	Console.WriteLine("Root: {0}",dir.Root);
}
```

你的输出结果可能会不同，但是你应该能看到类似下面的信息:

```
FullName: c:\Windows
Name: Windows
Parent:
Creation: 2015/11/2 11:32:51
Attributes: Directory
Root: c:\
```

### 使用 DirectoryInfo 类型枚举出文件

除了获取已存在目录的基本信息外，我们还能使用 DirectoryInfo 类型的一些方法来扩展当前的例子。首先，使用 GetFiles() 方法来获取 f:\test 目录下所有*.png文件的信息。

如果你机器没有 f:\test 目录，则修改代码来读取机器目录里的文件

```c#
static void DisplayImageFiles()
{
	DirectoryInfo dir = new DirectoryInfo(@"f:\test");
	// 获取所有*.png文件
	FileInfo[] imageFiles = dir.GetFiles("*.png", SearchOption.AllDirectories);

	// 我们找到多少图片
	Console.WriteLine("Found {0} *.png files\n",imageFiles.Length);

	// 输出每个文件的信息
	foreach (FileInfo f in imageFiles)
	{
		Console.WriteLine("File name: {0}",f.Name);
		Console.WriteLine("File size: {0}",f.Length);
		Console.WriteLine("Creation: {0}",f.CreationTime);
		Console.WriteLine("Attributes: {0}",f.Attributes);
	}
}
```

在调用 GetFiles()时指定了搜索选项(search option)，就可以查看所有子目录

### 使用 DirectoryInfo 类型创建子目录

我们能使用 DirectoryInfo.CreateSubdirectory() 方法以编程方式扩展目录结构。使用这个方法，可以建立一个子目录，也可以去一次建立多个嵌套子目录。

```c#
static void ModifyAppDirectory()
{
	DirectoryInfo dir = new DirectoryInfo(@"f:\test");

	// 在应用程序目录下创建\MyFolder
	dir.CreateSubdirectory("MyFolder");

	// 在应用程序目录下创建\MyFolder\Data
	dir.CreateSubdirectory(@"MyFolder2\Data");
}
```

如果从 Main() 调用这个方法并使用 Windows资源查看器来检查目录，会发现子目录已经成功创建。文件夹如果创建成功，它会返回表示新建项的 DirectoryInfo 类型。

```c#
static void ModifyAppDirectory()
{
	DirectoryInfo dir = new DirectoryInfo(@"f:\test");

	// 在应用程序目录下创建\MyFolder
	dir.CreateSubdirectory("MyFolder");

	// 捕获返回的 Directoryinfo类型
	DirectoryInfo myDataFolder = dir.CreateSubdirectory(@"MyFolder2\Data");

	// 在..\MyFolder2\Data上输出路径
	Console.WriteLine("New Folder is： {0}",myDataFolder);
}
```

运行成功后，输出创建的文件夹

```
New Folder is: f:\test\MyFolder2\Data
```

### 使用 Directory 类型

在实践过了 DirectoryInfo 以后，我们来研究 Directory 类型。Directory 的静态成员实现了由 DirectoryInfo 定义的实例级成员的大部分功能。Directory 成员返回的是字符串数据而不是强类型的 FileInfo 和 DirectoryInfo 对象。

```c#
static void FunWithDirectoryType()
{
	// 列出当前电脑的所有驱动器
	string[] drives = Directory.GetLogicalDrives();
	Console.WriteLine("Here are your drives:");
	foreach (string s in drives)
	{
		Console.WriteLine("--> {0} ",s);
	}

	// 删除签名建立的目录
	try
	{
		Directory.Delete(@"f:\MyFolder");

		// 第二个参数指定你师傅希望删除某个子目录
		Directory.Delete(@"f:\MyFolder2",true);
	}
	catch (IOException e)
	{
		Console.WriteLine(e.Message);
	}
}
```
